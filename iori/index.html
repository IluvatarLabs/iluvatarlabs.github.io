Iluvatar Open Research Initiative.

Open source science powered by Marvin. We run questions the field hasn't gotten to, in the open. Anyone can contribute.

An evolving open program for questions worth answering. Marvin runs the research. Anyone can contribute. Iluvatar takes no IP.

Project ideas come from the community. Anyone can propose a question worth running. The submitting researcher leads the resulting project with full attribution.
Critiques, suggestions, and feedback are public. Every project's methods, hypotheses, and uncertainty estimates are exposed as the work progresses. Anyone can challenge a methodology, flag a missing reference, or push back on a confidence tier. Discussions live on GitHub alongside the work.
Data is shared, found, and contributed. Projects run on public data by default. Anyone can point us at datasets we missed, contribute curated or processed data, or volunteer to run validation on a published hypothesis.
Marvin runs the research, in public. Marvin handles literature synthesis, data integration, analysis, and drafting. Iluvatar PhDs vet every output before it ships. Iterations land as pull requests against the project repo, sit for community review, then merge. Each draft leaves open questions for collaborators.

Contribute.

01 Critique or challenge a finding
Push back on an interpretation, flag a confounder we missed, or challenge a confidence classification. Every finding carries an explicit evidence tier. Tell us if you think we got one wrong. Open a critique on GitHub →
02 Contribute data
Point us at a dataset we missed, share curated or processed data, or contribute validation results from your own work. New data feeds the next iteration of Marvin's analysis. Suggest a dataset →
03 Suggest a new direction
Is there a question we should be asking that we haven't? A hypothesis Marvin should test? A method worth trying? Post it and Marvin will explore it in the next research cycle. Propose a direction →
04 Validate a hypothesis
Run a wet-lab experiment, a computational replication, or an independent analysis on one of our published findings. Validation results publish alongside the original work with full attribution. Browse open hypotheses →
05 Submit a new project idea
Propose a question you think Marvin should run. We review every submission and reply with specific feedback either way. Submit a question →
